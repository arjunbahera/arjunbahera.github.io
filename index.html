CASE FILE 02 // FINTECH PAYMENT GATEWAY
Context

A large-scale fintech payment gateway processing transactions across 12 markets. The authentication and authorization control plane spanned Microsoft B2C, Auth0, and custom identity federation.

Risk

Misconfigured identity federation paths created cross-tenant account takeover vectors. An exploited path would have exposed transaction authorization for millions of user accounts.

Approach

Conducted Zero Trust architecture review with STRIDE-based threat modeling. Mapped all token exchange paths, validated OAuth2/OIDC flows, and correlation-tested every trust boundary across the federated identity graph.

Outcome

Identified and remediated 3 critical auth-bypass paths before production deployment. Zero authentication-related findings in the subsequent SOC 2 Type II audit. Delivered board-ready risk memo within 10 business days.

CASE FILE 04 // CLOUD-NATIVE HEALTHCARE
Context

A cloud-native healthcare platform handling PHI across AWS. The codebase processed sensitive patient data through a microservice architecture with LLM-based diagnostic components under development.

Risk

LLM integration paths introduced novel prompt injection and data poisoning surfaces. A successful prompt injection could have exfiltrated PHI through the diagnostic pipeline, creating a HIPAA-reportable event.

Approach

Performed security risk analysis of LLM models against OWASP Top 10 for LLMs. Built agentic security workflows using LLM-based tooling for automated code review, correlating findings across SAST output from Checkmarx and SonarQube. Automated manual review with custom Semgrep rules.

Outcome

Time-to-triage reduced by 60% through automated CI/CD-integrated security scanning. LLM risk assessment delivered to the CISO and compliance officer for board presentation. No data exfiltration events in the 12-month post-assessment window.
